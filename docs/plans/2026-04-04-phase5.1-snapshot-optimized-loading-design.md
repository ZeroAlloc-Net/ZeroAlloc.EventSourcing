# Phase 5.1: Snapshot-Optimized Repository Loading — Design Document

**Date:** 2026-04-04  
**Status:** Approved  
**Approach:** Decorator Pattern

---

## Overview

Optimize aggregate loading by leveraging snapshots. Instead of replaying all events from stream start, load the latest snapshot and only replay events after that position. This reduces read amplification for long-lived aggregates.

**Goal:** Enable optional snapshot-based repository loading without modifying `AggregateRepository`.

---

## Architecture

### Core Principle: Decorator Pattern

Keep `AggregateRepository<TAggregate, TId>` **unchanged**. Create a decorator that wraps it:

```
User Code
  ↓
SnapshotCachingRepositoryDecorator<TAggregate, TId, TState>
  ↓
AggregateRepository<TAggregate, TId> (unchanged)
  ↓
IEventStore
```

**Benefits:**
- Zero breaking changes to core repository
- Snapshot logic is composable and optional
- Multiple strategies without conditionals
- Easy to test in isolation
- Follows Liskov Substitution Principle

## Components

### 1. ISnapshotLoadingStrategy

```csharp
public enum SnapshotLoadingStrategy
{
    /// Trust snapshot completely; replay events from snapshot position
    TrustSnapshot = 0,
    
    /// Validate snapshot position exists in event store before using
    ValidateAndReplay = 1,
    
    /// Ignore snapshot; always replay from start
    IgnoreSnapshot = 2
}
```

Users configure the strategy when composing the decorator.

### 2. SnapshotCachingRepositoryDecorator<TAggregate, TId, TState>

Wraps `IAggregateRepository<TAggregate, TId>`:

```csharp
public sealed class SnapshotCachingRepositoryDecorator<TAggregate, TId, TState>
    : IAggregateRepository<TAggregate, TId>
    where TAggregate : IAggregate
    where TId : struct
    where TState : struct, IAggregateState<TState>
{
    private readonly IAggregateRepository<TAggregate, TId> _innerRepository;
    private readonly ISnapshotStore<TState> _snapshotStore;
    private readonly SnapshotLoadingStrategy _strategy;
    private readonly Func<TAggregate, TState> _getState;
    private readonly Action<TAggregate, TState, StreamPosition> _setState;
    private readonly Func<TId, StreamId> _streamIdFactory;
    private readonly IEventStore _eventStore;

    public SnapshotCachingRepositoryDecorator(
        IAggregateRepository<TAggregate, TId> innerRepository,
        ISnapshotStore<TState> snapshotStore,
        SnapshotLoadingStrategy strategy,
        Func<TAggregate, TState> getState,
        Action<TAggregate, TState, StreamPosition> setState,
        Func<TId, StreamId> streamIdFactory,
        IEventStore eventStore);

    public async ValueTask<Result<TAggregate, StoreError>> LoadAsync(
        TId id, 
        CancellationToken ct = default)
    {
        var streamId = _streamIdFactory(id);
        
        // Try to load snapshot
        if (_strategy != SnapshotLoadingStrategy.IgnoreSnapshot)
        {
            var snapshot = await _snapshotStore.ReadAsync(streamId, ct);
            if (snapshot.HasValue)
            {
                // Snapshot found — restore state and replay from position
                var aggregate = await _innerRepository.LoadAsync(id, ct);
                _setState(aggregate, snapshot.Value.State, snapshot.Value.Position);
                
                // Load remaining events from snapshot position onward
                await ReplayFromPositionAsync(aggregate, streamId, snapshot.Value.Position.Next(), ct);
                
                return Result<TAggregate, StoreError>.Success(aggregate);
            }
        }
        
        // No snapshot or strategy ignores it — use inner repository (full replay)
        return await _innerRepository.LoadAsync(id, ct);
    }

    public ValueTask<Result<AppendResult, StoreError>> SaveAsync(
        TAggregate aggregate, 
        TId id, 
        CancellationToken ct = default)
    {
        // Delegate to inner repository
        return _innerRepository.SaveAsync(aggregate, id, ct);
    }

    private async Task ReplayFromPositionAsync(
        TAggregate aggregate,
        StreamId streamId,
        StreamPosition fromPosition,
        CancellationToken ct)
    {
        // Replay remaining events
        await foreach (var envelope in _eventStore.ReadAsync(streamId, fromPosition, ct))
        {
            aggregate.ApplyHistoric(envelope.Event, envelope.Position);
        }
    }
}
```

## Data Flow

```
LoadAsync(aggregateId)
  │
  ├─→ Check snapshot store for latest snapshot
  │    │
  │    ├─→ Snapshot found?
  │    │    │
  │    │    ├─ YES → Restore state to aggregate
  │    │    │        Replay events from (position + 1) onward
  │    │    │        Return loaded aggregate
  │    │    │
  │    │    └─ NO → Fall through to inner repository
  │    │
  │    └─→ Strategy = IgnoreSnapshot?
  │         └─ YES → Skip snapshot check, use inner repository
  │
  └─→ Inner repository: replay all events from start
```

---

## Error Handling

**Snapshot Load Failure:**
- Catch exception, log warning
- Fall back to full event replay via inner repository
- Graceful degradation — service remains available

**Invalid Snapshot Position:**
- Strategy-dependent:
  - `TrustSnapshot`: Use as-is (performance over safety)
  - `ValidateAndReplay`: Check position exists in event store; if not, replay from start
  - `IgnoreSnapshot`: Skip snapshot entirely

**Corrupted Snapshot State:**
- Deserialization error → fall back to full replay
- Allow user override via exception handler

## Testing Strategy

**Unit Tests:**
- Decorator without snapshot → delegates to inner repository
- Decorator with snapshot → loads snapshot, replays from position
- Strategy configurations (Trust, Validate, Ignore)
- Error cases (snapshot not found, load fails, invalid position)

**Integration Tests:**
- With InMemorySnapshotStore: verify snapshot restore works
- With real aggregates: Order, Invoice, etc.
- Snapshot consistency: verify replayed events match full replay result

## Configuration Example

```csharp
var innerRepo = new AggregateRepository<Order, OrderId>(...);
var snapshotStore = new InMemorySnapshotStore<OrderState>();

var snapshotRepo = new SnapshotCachingRepositoryDecorator<Order, OrderId, OrderState>(
    innerRepository: innerRepo,
    snapshotStore: snapshotStore,
    strategy: SnapshotLoadingStrategy.ValidateAndReplay,
    getState: order => order.State,
    setState: (order, state, pos) => { /* restore state */ },
    streamIdFactory: id => new StreamId($"order-{id.Value}"),
    eventStore: eventStore
);

// Use like normal repository
var result = await snapshotRepo.LoadAsync(new OrderId(123));
```
